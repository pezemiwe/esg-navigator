# Climate Physical Risk Assessment Engine v3.0 — Copilot Build Guide

## Strategy

This monolithic 6,200-line Python script has been decomposed into two parts:

1. **16 JSON data files** (135KB) — all lookup tables, extracted verbatim. These go into `src/data/` and are **never regenerated by Copilot**. They are the actuarial backbone of the engine.
2. **This prompt guide** — module-by-module specifications for Copilot to build the logic layer against those JSON files.

### PROMPT 1: Data Layer + Loader Utilities

```
Create src/data/__init__.py that loads all JSON files from src/data/ into module-level constants.
Each JSON file should be loaded once at import time and exposed as a named constant.

The JSON files are already present in src/data/. Here is what each contains:

- asset_types.json → list of 47 string asset type names
- all_21_risks.json → list of 21 dicts, each with keys: risk (str), category (str), sub_category (str)
- ef_table.json → dict[asset_type → dict[hazard_rating → float]]. 47 asset types × 6 ratings (Extreme/Very High/High/Medium/Low/Negligible). Values are exposure fractions 0.0–1.0.
- inherent_vulnerability.json → dict[risk_name → dict[asset_type → float]]. 21 risks × 47 asset types. Values 0.0–1.0.
- sbra_base_rrf.json → dict[sector_name → dict[asset_type → float]]. 25 sectors × 47 asset types. Values are base Resilience Reduction Factors.
- sectors.json → dict[sector_number_str → {name: str, subsectors: list[str]}]. 25 sectors.
- subsector_adj.json → dict[subsector_name → float]. Adjustment values to add to SBRA base RRF. Has "__default__" key = 0.0.
- sector_risk_modifiers.json → dict[sector_name → dict[risk_name → float]]. 8 sectors with per-risk adjustments.
- default_risk_modifiers.json → dict[risk_name → float]. Fallback per-risk adjustment for sectors not in sector_risk_modifiers.
- alra_measure_contributions.json → dict[measure_name → dict[risk_name → float]]. 30 resilience measures, each with per-risk contributions.
- annual_probability.json → dict[frequency_label → float]. 6 labels: Almost Certain=0.90, Very Frequent=0.50, Frequent=0.20, Occasional=0.10, Rare=0.04, Very Rare=0.01.
- rating_order.json → dict[rating_label → int]. Negligible=1 through Extreme=6.
- revenue_disruption.json → dict[asset_type → dict[hazard_rating → [rdf, downtime_days]]]. 47 types × 6 ratings. Each value is a 2-element array [disruption_fraction_per_day, downtime_days_int].
- rdt_default.json → same structure as one entry in revenue_disruption, used as fallback.
- opex_uplift.json → dict[risk_name → dict[asset_type → float]]. 6 chronic risks × 47 asset types.
- opex_chronic_risks.json → list of 6 risk names that have OPEX impact.
- response_rules.json → dict[hazard_rating → {strategy, priority, timeframe, residual_reduction}]. 6 ratings.
- response_actions.json → dict[risk_name → list[str]]. 21 risks, each with 3 recommended actions.
- monitoring_config.json → dict[risk_name → {kpi, trigger, review_freq, data_source, owner_role, owner_name}]. 21 risks.

Export all as UPPER_SNAKE_CASE constants: EF_TABLE, INHERENT_VULNERABILITY, SBRA_BASE_RRF, etc.
Add type hints. Include a get_data_path() helper for resolving JSON paths relative to this file.
```

### PROMPT 3: API Clients

```
Create src/apis/ with the following modules. Each function should have retry logic (max 3 attempts), proper timeout handling (120s default), and return None on failure. Use the requests library.

geocoding.py:
- extract_city_region(address: str) → str — extract last 2 comma-separated parts of address
- geocode(address: str) → tuple[float, float] | None — use Nominatim (https://nominatim.openstreetmap.org/search) with format=json, limit=1. Return (lat, lon) or None.
- get_coordinates(asset_name: str, address: str) → tuple[float|None, float|None, str] — try full address first, then city/region fallback, then asset_name only. Return (lat, lon, confidence) where confidence is "Exact", "City-level", "Name-only", or "Failed".

nasa_power.py:
- fetch_nasa_daily(lat: float, lon: float) → pd.DataFrame | None — GET https://power.larc.nasa.gov/api/temporal/daily/point with params: parameters=PRECTOTCORR, community=RE, start=19900101, end=20231231, format=JSON. Return DataFrame with columns [date, rainfall_mm].
- fetch_nasa_monthly(lat: float, lon: float) → pd.DataFrame | None — same endpoint but temporal/monthly, parameters=PRECTOTCORR, start=1990, end=2023. Return DataFrame with [year, month, rainfall_mm].
- fetch_nasa_heat(lat: float, lon: float) → pd.DataFrame | None — daily endpoint, parameters=T2M_MAX,RH2M, start=19940101, end=20231231. Return DataFrame with [max_temp, humidity].

chirps.py:
- fetch_chirps(lat: float, lon: float) → pd.DataFrame | None — GET https://data.chc.ucsb.edu/products/CHIRPS-2.0/global_daily/csvs/ (construct URL for nearest point). Parse CSV into DataFrame with [date, rainfall_mm]. If CHIRPS fails, return None (caller falls back to NASA POWER).

elevation.py:
- get_elevation_srtm(lat: float, lon: float) → float | None — GET https://api.open-elevation.com/api/v1/lookup?locations={lat},{lon}. Return elevation in metres.
- check_if_coastal(lat: float, lon: float) → bool — use Nominatim reverse geocode, then check if any point within ~1km is water, OR use elevation < 10m as coastal proxy.
- get_coastal_flood_depth(lat: float, lon: float) → float — estimate flood depth based on elevation and coastal proximity.

ibtracs.py:
- load_ibtracs(method: str, file_path: str | None = None) → pd.DataFrame | None — if method=="STREAM", download from https://www.ncei.noaa.gov/data/international-best-track-archive-for-climate-stewardship-ibtracs/v04r01/access/csv/ibtracs.ALL.list.v04r01.csv. If method=="LOCAL", read from file_path.
- query_ibtracs(df: pd.DataFrame, lat: float, lon: float, radius_km: float = 500) → dict — find cyclones within radius using haversine distance. Return stats: count, max_wind_kt, closest_km, most_recent_year.

usgs.py:
- fetch_earthquakes(lat: float, lon: float, radius_km: float = 200) → dict | None — GET https://earthquake.usgs.gov/fdsnws/event/1/query with format=geojson, latitude, longitude, maxradiuskm, starttime=1990-01-01. Return stats: count, max_magnitude, closest_km.

General:
- haversine_km(lat1, lon1, lat2, lon2) → float — standard haversine formula returning distance in km. Put this in a shared utils module.

All API clients should log to stdout with the pattern: "  [{RiskName}] Fetching {Source}..." and "  [{RiskName}] Failed: {error}".
```

### PROMPT 4: Hazard Assessment Functions

```
Create src/hazards/base.py with a base class and individual hazard modules.

Base class pattern — each assess_* function takes (asset_name, lat, lon, confidence, mc) and returns a result dict (via make_result) or None on failure. All raw scores are computed on a 1–3 scale then scaled to mc['size'] using scale_score().

Create these specific implementations (refer to src/apis/ for data fetching):

heat.py — assess_extreme_heat:
  API: NASA POWER daily (T2M_MAX, RH2M, 1994-2023)
  Intensity: effective_temp = max_temp + humidity_adjustment (if humidity > 40%: (humidity-40)*0.1)
    raw_i: <38°C=1, <42°C=2, ≥42°C=3
  Frequency: days_above_38 per year (averaged over record)
    raw_f: <10=1, <30=2, ≥30=3
  Extra fields: max_recorded_temp_celsius, mean_humidity_pct, effective_heat_index_max, days_above_38_per_year, days_above_40_per_year

drought.py — assess_drought:
  API: CHIRPS daily → fallback NASA POWER monthly. Need ≥5 complete years.
  Compute SPI (Standardised Precipitation Index) = (annual_rainfall - mean) / std
  Intensity: min_spi: ≥-1.0=1, ≥-1.5=2, <-1.5=3
  Frequency: pct of years with SPI < -1.0: <10%=1, <30%=2, ≥30%=3
  Extra: minimum_spi, worst_drought_year, drought_years_count, total_years_analysed, drought_frequency_pct

rainfall.py — assess_heavy_rainfall:
  API: CHIRPS daily → fallback NASA POWER daily
  Intensity: max single-day rainfall (rx1day): <50mm=1, <100mm=2, ≥100mm=3
  Frequency: days_above_50mm per year: <2=1, <5=2, ≥5=3
  Extra: rx1day_mm, days_above_50mm_per_year, days_above_100mm_per_year

harmattan.py — assess_harmattan:
  API: NASA POWER daily (T2M_MAX, RH2M, 1994-2023)
  Compute harmattan_days = days where humidity < 20% AND temp > 35°C during Nov-Mar
  Intensity: harmattan severity index based on consecutive dry days
    raw_i: <30 days/yr=1, <60=2, ≥60=3
  Frequency: percentage of years with >30 harmattan days: <20%=1, <50%=2, ≥50%=3
  Extra: harmattan_days_per_year, mean_dry_season_humidity, dust_severity_index

thunderstorms.py — assess_thunderstorms:
  API: NASA POWER (lightning proxy via PRECTOTCORR > 20mm days during convective months Apr-Oct)
  Intensity: estimated thunderstorm days/year: <30=1, <60=2, ≥60=3
  Frequency: same metric: <30=1, <60=2, ≥60=3

flooding.py — assess_flash_flooding, assess_river_flooding, assess_coastal_flooding, assess_storm_surge, assess_groundwater_flooding:
  All use combinations of: CHIRPS/NASA rainfall data, elevation (SRTM), coastal proximity check.
  Flash flooding intensity: based on max hourly rainfall intensity proxy (rx1day)
  River flooding: based on elevation + proximity to river (low elevation + high rainfall = higher risk)
  Coastal flooding: check_if_coastal + elevation + tidal data
  Storm surge: coastal proximity + cyclone exposure
  Groundwater: elevation + rainfall patterns
  Each follows the raw_i (1-3) → scale_score → make_result pattern.

coastal.py — assess_coastal_erosion, assess_sea_level_rise:
  Erosion: coastal proximity + elevation + soil type proxy
  Sea level rise: elevation + IPCC projection (3.3mm/yr baseline, accelerating)

geological.py — assess_earthquakes, assess_volcanic_eruptions, assess_tsunamis, assess_landslides:
  Earthquakes: USGS API query within 200km. Intensity: max_magnitude <4=1, <6=2, ≥6=3. Frequency: events/decade.
  Volcanic: distance to nearest Holocene volcano (use hardcoded major volcano coordinates or Smithsonian GVP).
  Tsunamis: coastal + seismic exposure
  Landslides: elevation gradient + rainfall + soil

fire.py — assess_wildfire:
  Proxy: combine high temp days + low humidity days + vegetation (assume medium vegetation as default)
  Intensity/frequency on 1-3 scale

water.py — assess_water_scarcity, assess_glacial_retreat:
  Water scarcity: mean annual rainfall vs aridity threshold. <400mm=high, <600mm=medium, ≥600mm=low
  Glacial retreat: latitude-based proxy (only relevant >60° or high altitude)

desertification.py — assess_desertification:
  Rainfall trend + aridity index

cyclones.py — assess_tropical_cyclones:
  Uses IBTrACS data. Queries storms within 500km.
  Intensity: max_wind_kt: <64kt=1, <96kt=2, ≥96kt=3
  Frequency: storms per decade: <1=1, <3=2, ≥3=3
  Returns (result, ibtracs_df) tuple so IBTrACS can be reused across assets.

Register all assessors in src/hazards/__init__.py as a dict[risk_name → callable].
```

### PROMPT 5: Enrichment Pipeline (Steps 4, 5, 6)

```
Create src/engine/exposure.py, vulnerability.py, financial.py, and enrichment.py.

exposure.py:
- get_ef(hazard_rating: str, asset_type: str) → float
  Look up EF_TABLE[asset_type][hazard_rating]. Default to EF_TABLE["Office Building"] if asset type not found. Default 0.10 if rating not found.

vulnerability.py:
- get_inherent_vulnerability(risk: str, asset_type: str) → float
  Look up INHERENT_VULNERABILITY[risk][asset_type]. Default 0.50 if not found.

- get_sbra_rrf(risk, asset_type, sector_name, subsector_name, cache: dict) → float
  Formula: base_rrf + subsector_adj + sector_risk_modifier
  - base_rrf = SBRA_BASE_RRF[sector_name][asset_type], fallback to "Government & Public Sector", fallback 0.25
  - subsector_adj = SUBSECTOR_ADJ[subsector_name], fallback SUBSECTOR_ADJ["__default__"] = 0.0
  - risk_modifier = SECTOR_RISK_MODIFIERS[sector_name][risk], fallback DEFAULT_RISK_MODIFIERS[risk], fallback 0.0
  Clamp final to [0.10, 0.70]. Cache by key "{sector}|{subsector}|{asset_type}|{risk}".

- get_alra_rrf(asset_name, risk, alra_register: dict) → float | None
  Sum contributions from ALRA_MEASURE_CONTRIBUTIONS for each measure the asset has (from alra_register). Cap at 0.70. Return None if asset not in register.

financial.py:
- get_rdf_and_downtime(hazard_rating: str, asset_type: str) → tuple[float, int]
  Look up REVENUE_DISRUPTION_TABLE[asset_type][hazard_rating] → [rdf, downtime_days]. Fallback to RDT_DEFAULT.

- get_opex_uplift(risk: str, asset_type: str) → float
  Look up OPEX_UPLIFT_TABLE[risk][asset_type]. Return 0.0 if risk not in table (only 6 chronic risks have OPEX impact).

enrichment.py — enrich_results(all_results, mc, asset_register, alra_register, sector_name, subsector_name, usd_rate, vuln_method) → list[dict]:
  For each result in all_results:
    1. Step 4 — Exposure:
       ef = get_ef(hazard_rating, asset_type)
       exposed_value = asset_value × ef
       annual_prob = ANNUAL_PROBABILITY[frequency_label] (default 0.10)

    2. Step 5 — Vulnerability:
       inherent_v = get_inherent_vulnerability(risk, asset_type)
       sbra_rrf = get_sbra_rrf(...)
       sbra_net_v = inherent_v × (1 - sbra_rrf)
       If vuln_method in [ALRA, BOTH]:
         alra_rrf = get_alra_rrf(asset_name, risk, alra_register)
         If alra_rrf is None → fallback to sbra values
         alra_net_v = inherent_v × (1 - alra_rrf)
       If vuln_method == SBRA: alra values = sbra values

    3. Step 6 — Risk Score:
       v_scaled = clamp(round(net_v × max_size), 1, max_size)
       raw_score = i_score × f_score × v_scaled
       norm_score = (raw_score / max_size³) × 100

    4. Step 6 — Asset Financial Loss:
       SSL = asset_value × ef × net_v
       EAL = SSL × annual_probability
       (Compute for both SBRA and ALRA, in both NGN and USD)

    5. Step 6 — Revenue Financial Loss:
       rev_fraction = rdf × (downtime_days / 365)
       rev_SSL = annual_revenue × rev_fraction × net_v
       rev_EAL = rev_SSL × annual_probability

    6. Step 6 — OPEX Uplift:
       opex_EAL = annual_opex × uplift_factor × annual_probability

    7. Step 6 — Combined:
       total_EAL = asset_EAL + revenue_EAL + opex_EAL

  Output fields per record (merge with original result dict):
    asset_value_ngn, asset_value_usd, asset_type, exposure_factor, exposed_value_ngn, exposed_value_usd,
    inherent_vulnerability, sbra_rrf, sbra_net_vulnerability, alra_rrf, alra_net_vulnerability,
    annual_probability, risk_score_raw_sbra, risk_score_norm_sbra, risk_score_raw_alra, risk_score_norm_alra,
    ssl_sbra_ngn, ssl_sbra_usd, ssl_alra_ngn, ssl_alra_usd, eal_sbra_ngn, eal_sbra_usd, eal_alra_ngn, eal_alra_usd,
    annual_revenue_ngn, rev_disruption_factor, rev_downtime_days, rev_ssl_sbra_ngn, rev_ssl_alra_ngn, rev_ssl_alra_usd,
    rev_eal_sbra_ngn, rev_eal_alra_ngn, rev_eal_alra_usd,
    annual_opex_ngn, opex_uplift_factor, opex_eal_ngn, opex_eal_usd,
    total_eal_alra_ngn, total_eal_sbra_ngn, total_eal_alra_usd

Add full type hints and docstrings.
```

### PROMPT 6: Steps 7, 8, 9

```
Create src/engine/evaluation.py, response.py, monitoring.py.

evaluation.py:
- print_risk_heatmap_console(enriched_results, mc, cfg) → None
  Print a text-based risk heatmap to stdout. Group by hazard_rating, show counts per rating level.

response.py:
- calculate_residual_risk(risk_score_norm: float, reduction_factor: float) → tuple[float, str]
  residual_score = risk_score_norm × (1 - reduction_factor), rounded to 1dp.
  Residual rating: ≥80=Extreme, ≥60=Very High, ≥40=High, ≥20=Medium, ≥5=Low, else Negligible.

- add_step_8_response(enriched_results: list[dict]) → list[dict]
  For each result:
    Look up RESPONSE_RULES[hazard_rating] → {strategy, priority, timeframe, residual_reduction}
    Look up RESPONSE_ACTIONS[risk] → list of 3 action strings (default: ["Conduct detailed risk assessment", "Implement appropriate controls", "Monitor and review annually"])
    Calculate residual risk using risk_score_norm_alra and residual_reduction.
    Append fields: response_strategy, response_priority, response_timeframe, response_actions (pipe-separated string), residual_reduction_pct, residual_risk_score, residual_risk_rating.

monitoring.py:
```

## How to Use This Guide

1. **Create the repo** with the folder structure above.
2. **Copy all 16 JSON files** from the provided `src/data/` into your repo.
3. **Create `requirements.txt`**: `pandas`, `numpy`, `requests`.

5. **Run the test suite** after Prompt 9 to validate data integrity and formulas.
6. **Functional test**: run `python src/main.py` and compare output against the original monolith.
