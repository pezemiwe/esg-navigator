# Transition Risk Assessment Engine v1.0 — Copilot Build Guide

## What This Engine Does

Measures the financial risk arising from the global shift to a low-carbon economy for any organisation, in any country, across any sector.

**Four transition risk drivers:**
1. Policy & Regulation — carbon pricing, NDCs, phase-out mandates
2. Technology Disruption — revenue model vulnerability to substitution
3. Market & Sentiment — investor ESG screening, capital cost repricing
4. Legal & Liability — litigation exposure, disclosure liability

**Three NGFS scenarios × three time horizons:**
- Orderly (Net Zero 2050) · Disorderly (Delayed Action) · Hot House World
- Short (1–3yr) · Medium (3–10yr) · Long (10–30yr)

**Outputs per organisation:**
- Transition Risk Score (0–100) and Rating (Negligible → Extreme)
- Driver scores (Policy, Technology, Market, Legal) each 0–100
- ΔPD (basis points) under each scenario × horizon
- Stranded Asset Discount (fraction of asset value)
- Revenue Erosion % under each scenario × horizon
- Transition EAL (expected annual loss in local currency + USD)
- Physical+Transition combined EAL connector

## Strategy

Same as the physical risk engine: data tables extracted to JSON (never regenerated by Copilot), logic built module-by-module.

**Extracted JSON files (6 files, ~30KB):**
- country_policy_db.json — 102 countries × 6 policy variables
- country_fallbacks.json — 5 income-group fallback profiles
- sector_db.json — 25 sectors × 10 sector variables
- ngfs_scenarios.json — 3 scenarios with full carbon price/tech/market/legal parameters
- time_horizons.json — 3 horizons with mid-year and weight
- rating_icons.json — display icons per rating

### PROMPT 3: Four Driver Scoring Functions

```
Create src/drivers/policy.py, technology.py, market.py, legal.py, composite.py.

All driver functions take (sector_data, country_data, org_profile, scenario, horizon) → float (0–100).

policy.py — score_policy_driver:
  1. Effective carbon price = max(base_price × scenario_multiplier, scenario_floor)
     - base_price = max(country_data["carbon_price"], 1)
     - multiplier = NGFS_SCENARIOS[scenario][f"carbon_price_{horizon.lower()}"][income_group]
     - floor = NGFS_SCENARIOS[scenario][f"carbon_price_floor_{horizon.lower()}"][income_group]
  2. carbon_cost_pct_revenue = (sector.carbon_intensity_tco2_musd × eff_price) / 1_000_000 × 100
  3. policy_sens = sector.policy_sensitivity / 5.0
  4. country_factor = (ndc/5) × (credibility/5) × scenario.policy_delivery_pct
  5. rev_exposure = org.revenue_carbon_fraction (×0.7 if has_transition_plan)
  6. raw = carbon_cost_pct_revenue × policy_sens × country_factor × rev_exposure
  7. score = min(100, raw × 4)

technology.py — score_technology_driver:
  1. tech_vuln = sector.tech_disruption / 5.0
  2. adoption_speed = NGFS_SCENARIOS[scenario][f"tech_speed_{horizon.lower()}"]
  3. country_speed = country.energy_transition / 5.0
  4. phase_out_year: use "orderly" key for Orderly scenario, "disorderly" for others
     years_remaining = max(1, phase_out_year - 2025)
     time_pressure = clamp((TIME_HORIZONS[horizon].mid_year - 2025) / years_remaining, 0, 1)
  5. tech_dep = org.technology_dependency × (1 - org.low_carbon_revenue_fraction × 0.5)
  6. raw = tech_vuln × adoption_speed × country_speed × time_pressure × tech_dep × 200
  7. score = min(100, raw)

market.py — score_market_driver:
  1. mkt_sens = sector.market_sensitivity / 5.0
  2. premium_bps = NGFS_SCENARIOS[scenario][f"market_premium_bps_{horizon.lower()}"]
     premium_impact = min(1.0, premium_bps / 300.0)
  3. intl_exposure = org.international_capital_exposure
     listed = 1.2 if org.is_listed else 1.0
  4. supply_chain = (sector.supply_chain_exposure / 5.0) × org.supply_chain_carbon_exposure
  5. income_factor = {"H": 1.0, "UM": 0.8, "LM": 0.6, "L": 0.4}[income]
  6. raw = mkt_sens × premium_impact × intl_exposure × listed × income_factor × 100
     raw += supply_chain × 20
  7. score = min(100, raw)

legal.py — score_legal_driver:
  1. legal_base = sector.legal_exposure / 5.0
  2. legal_mult = NGFS_SCENARIOS[scenario].legal_multiplier
  3. legal_jurisdiction = (income in ["H","UM"]) × 0.6 + (credibility >= 3) × 0.4
  4. disclosure_factor = 1.0 - (org.disclosure_quality × 0.5)
  5. fossil_premium = 1.5 if sector.carbon_intensity > 800 else 1.0
  6. horizon_factor = {"Short": 0.5, "Medium": 1.0, "Long": 1.5}[horizon]
  7. raw = legal_base × legal_mult × legal_jurisdiction × disclosure_factor × fossil_premium × horizon_factor × 100
  8. score = min(100, raw)

composite.py:
- compute_transition_risk_score(policy, technology, market, legal) → float
  Weights: Policy 35%, Technology 30%, Market 25%, Legal 10%
  score = policy×0.35 + technology×0.30 + market×0.25 + legal×0.10

- score_to_rating(score: float) → str
  <10=Negligible, <25=Low, <45=Medium, <65=High, <80=Very High, ≥80=Extreme
```

### PROMPT 4: Credit Risk Parameters

```
Create src/credit/delta_pd.py, stranded_assets.py, revenue_erosion.py.

delta_pd.py — compute_delta_pd(transition_score, scenario, horizon, country_data, org_profile) → float (basis points):
  1. Base ΔPD from score band (ECB/BoE calibrated):
     <10→0, <25→8, <40→22, <55→55, <70→110, <85→195, ≥85→320
  2. scenario_mult: Orderly=0.7, Disorderly=1.8, Hot House World=0.3
  3. horizon_mult: Short=0.4, Medium=1.0, Long=1.8
  4. income_adj: H=1.0, UM=0.85, LM=0.65, L=0.45
  5. pd_adj = max(0.3, 1.0 - (current_pd_bps / 2000))
  6. delta_pd = base × scenario_mult × horizon_mult × income_adj × pd_adj

stranded_assets.py — compute_stranded_asset_discount(sector_data, country_data, scenario, horizon, org_profile) → float (0.0–0.80):
  - If sector.stranded_asset_risk <= 1 → return 0.0
  - If scenario == "Hot House World" → return 0.0
  - base_fraction = (stranded_asset_risk - 1) / 4.0 × 0.60
  - phase_out_year from sector (orderly key for Orderly, disorderly for Disorderly)
  - beyond_fraction = max(0, (2025 + remaining_life - phase_out_year)) / max(1, remaining_life)
  - credibility_factor = (credibility × policy_delivery_pct) / 5.0
  - horizon_factor: Short=0.2, Medium=0.6, Long=1.0
  - discount = base × beyond × credibility × horizon
  - If carbon_intensity > 1000: discount ×= 1.4
  - Clamp to [0.0, 0.80]

revenue_erosion.py — compute_revenue_erosion(sector_data, country_data, scenario, horizon, org_profile) → float (percentage 0–80):
  - tech_erosion = (tech_disruption/5) × tech_speed × min(1, horizon_years/15) × 0.35
  - mkt_erosion = (market_sensitivity/5) × (premium_bps/300) × min(1, horizon_years/20) × 0.15
  - low_carbon_offset = org.low_carbon_revenue_fraction × 0.5
  - total = max(0, (tech + mkt) × (1 - offset))
  - Scenario adjustments: Orderly ×0.7, Disorderly+Medium ×1.8, Hot House ×0.2
  - Clamp to 80%, return as percentage
  - horizon_years: Short=2, Medium=7, Long=20
```

### PROMPT 5: Transition EAL

```
Create src/financial/transition_eal.py.

compute_transition_eal(org_profile, sector_data, country_data, transition_score, revenue_erosion_pct, stranded_discount, scenario, horizon, usd_rate) → dict:

  Three components:

  1. Annual probability of transition shock:
     base_prob lookup:
       Orderly:    Short=0.05, Medium=0.12, Long=0.20
       Disorderly: Short=0.04, Medium=0.25, Long=0.35
       Hot House:  Short=0.01, Medium=0.03, Long=0.06
     annual_prob = base_prob × (transition_score / 100)

  2. Revenue Erosion EAL = annual_revenue × (revenue_erosion_pct / 100) × annual_prob

  3. OPEX Compliance EAL:
     Effective carbon price (same formula as policy driver):
       base_cp = max(country.carbon_price, 1)
       eff_cp = max(base_cp × multiplier, floor) using scenario/horizon/income
     revenue_musd = annual_revenue / (usd_rate × 1_000_000)
     carbon_levy_local = sector.carbon_intensity × eff_cp × revenue_musd × usd_rate
     opex_eal = (annual_opex × 0.05 + carbon_levy_local) × annual_prob

  4. Asset Stranding EAL:
     stranding_annual = (asset_value × stranded_discount) / remaining_life
     strand_eal = stranding_annual × annual_prob

  5. total_eal = rev_eal + opex_eal + strand_eal

  Return dict:
    revenue_erosion_eal_local, opex_compliance_eal_local, stranding_eal_local,
    transition_eal_local, transition_eal_usd (= total / usd_rate), annual_probability
```

### PROMPT 6: Assessment Orchestration

```
Create src/engine/assess.py and connector.py.

assess.py:
- assess_organisation(org_profile: dict, usd_rate: float = 1.0) → dict
  1. Look up sector_data and country_data
  2. Build results dict with org metadata + sector_carbon_intensity + country fields
  3. For each scenario in [Orderly, Disorderly, Hot House World]:
     For each horizon in [Short, Medium, Long]:
       - Compute 4 driver scores
       - Compute composite score + rating
       - Compute delta_pd, stranded_discount, revenue_erosion
       - Compute transition_eal
       - Store all in results["scenarios"][scenario][horizon]
  Return results

- assess_portfolio(org_profiles: list[dict], usd_rate: float) → list[dict]
  Loop over orgs, call assess_organisation for each, print progress.

connector.py:
- combine_with_physical(physical_results_df: pd.DataFrame, transition_results: list[dict]) → pd.DataFrame
  Build lookup: org_name → worst-case transition EAL/ΔPD/rating across all scenarios×horizons.
  Add columns to physical df: transition_eal_local, transition_dpd_bps, transition_risk_rating, combined_climate_eal_local (= physical total + transition).
```
